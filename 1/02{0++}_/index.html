Hexo

最近心血来潮，基于github page部署了个hexo博客系统。之后是一些使用排坑记录。

安装

hexo安装，依赖于nodejs,npm，首先需要装好对应的环境。安装好node后，执行

sudo npm install -g hexo-cli

hexo安装好后，执行命令初始化hexo博客

hexo init blog
cd blog
npm install

完成后blog下结构如下

├── _config.yml (hexo基础配置文件)
├── package.json
├── package-lock.json
├── node_modules(nodejs 相关目录)
├── source
|   └── _posts （存放博客文件）
└── themes(存放主题文件)

常用hexo命令

hexo clean #清理目录
hexo g #重新生成文件
hexo s #本地启动临时服务
hexo d #部署
hexo clean|hexo g|hexo s # 本地启动预览服务

必须配置

hexo初步安装完成后，还需要打开github配置，资源文件夹，etc.

打开_config.yaml_,对应行按如下进行改动

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: 'git'
  repo: https://github.com/Lubbock/Lubbock.github.io.git
  branch: master

使用Next主题

git clone https://github.com/iissnan/hexo-theme-next themes/next

这时候，如果使用hexo -s进行预览会发现categories和tags都是报错的，需要初始化tags,categories

vim source/categories/index.md
---
type: "categories"
layout: "categories"
---
vim source/tags/index.md
---
type: "tags"
layout: "tags"
---

新建about页面

hexo n about

这样子，一个基础的hexo配置已经算完成了。

hexo next 美化
