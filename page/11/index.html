简答题

  1. 简要论述CISAW软件安全开发模型的核心思想

    CISAW软件安全开发模型与CISAW信息安全保障模型一脉相承，通过实现 程序，数据和文档3个实体对象全生命周期的可用性，完整性,机密性,不可否认性等若干安全属性来支撑“业务”的正常进行，并在实现安全属性的过程中关注需求分析，设计，编码,测试,运行5个环节的技术手段和措施，模型中为实现上述对象的安全属性及5个环节需要充足的人力，财务，技术，信息资源提供保障，而以上的各类对象，环节，资源都必须进行综合有效的管理。

    1. 简述用户数据保护功能包含那几个功能类以及作用

    用户数据保护功能包含了访问控制，数据流控制，数据鉴别，系统内部传送，残余信息保护，回退，存储数据完整性共七个功能类，这些功能类规定了一些与用户数据保护相关的安全功能和安全策略

    1. 如何通过软件安全功能对用户数据进行保护

    为了保证软件安全，需要对输入，输出，和存储中的用户数据进行保护，而这种保护由用户数据保护功能来实现，具体来说，用户数据保护功能包含了访问控制，数据流控制，数据鉴别，系统内部传送，残余信息保护，回退，存储数据完整性`共七个功能类，这些功能类规定了一些与用户数据保护相关的安全功能和安全策略，其中：

    1. 访问控制类主要设计主题和课题间的操作权限
    2. 数据流控制主要处理系统和用户间，用户和用户间数据传输过程中的安全需求
    3. 数据鉴别主要保证存储数据的有效性
    4. 系统内部传送功能主要保护数据在传输过程中的安全可靠
    5. 残余信息保护主要为那些已被逻辑上删除或释放，但仍可能残存在系统中的信息提供保护功能
    6. 回退是未来保证用户数据的完整性
    7. 存储数据完整性则是为了让系统保证其存储的所有数据没有遭到破坏

Hexo

最近心血来潮，基于github page部署了个hexo博客系统。之后是一些使用排坑记录。

安装

hexo安装，依赖于nodejs,npm，首先需要装好对应的环境。安装好node后，执行

sudo npm install -g hexo-cli

hexo安装好后，执行命令初始化hexo博客

hexo init blog
cd blog
npm install

完成后blog下结构如下

├── _config.yml (hexo基础配置文件)
├── package.json
├── package-lock.json
├── node_modules(nodejs 相关目录)
├── source
|   └── _posts （存放博客文件）
└── themes(存放主题文件)

常用hexo命令

hexo clean #清理目录
hexo g #重新生成文件
hexo s #本地启动临时服务
hexo d #部署
hexo clean|hexo g|hexo s # 本地启动预览服务

必须配置

hexo初步安装完成后，还需要打开github配置，资源文件夹，etc.

打开_config.yaml_,对应行按如下进行改动

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: 'git'

  branch: master

LockFree

传统共享数据访问方式采用同步原语（临界区,锁,条件变量)等达到共享数据的安全访问，同步操作很容易成为并行程序中的瓶颈。

  • 同步原语是操作系统的内核对象，调用该原语会带来上下文切换（用户态换到内核态)
  • 杜绝并行操作，一个线程在访问共享数据的时候，其他多个线程必须在排队等待

于是，人们开始研究对共享数据进行并发访问的数据结构和算法

  • Transactional memory 事务性内存
  • Fine-grained algorithms 细粒度苏阿案发
  • Lock-free data structures 无锁数据结构
